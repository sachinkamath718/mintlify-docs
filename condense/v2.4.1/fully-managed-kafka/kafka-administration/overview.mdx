Condense offers a fully managed Apache Kafka service that simplifies the deployment, operation, and administration of Kafka clusters. It automates infrastructure provisioning, cluster scaling, maintenance, monitoring, and security, allowing engineering teams to focus on building and operating real-time data streaming applications instead of managing Kafka infrastructure.

Kafka Administration extends these capabilities by providing a centralized interface for managing Kafka resources and performing administrative operations directly from Condense. Administrators can manage topics, monitor consumer groups, configure users and access control, and perform operational tasks through an intuitive user interface or Kafka Management APIs.

Whether managing a development environment or a production deployment, Kafka Administration provides the tools required to administer Kafka clusters securely and efficiently.

## Key Capabilities

Kafka Administration enables administrators to:

- Create, view, update, and delete Kafka topics.
- Monitor and manage Kafka consumer groups.
- Manage Kafka users and assign administrative roles.
- Configure Access Control Lists (ACLs) for Kafka resources.
- Retrieve cluster metadata and topic information.
- Update topic configurations.
- Manage consumer group offsets.
- Automate Kafka administrative operations using REST APIs.

## Kafka Management APIs

Condense exposes Kafka Management APIs that enable applications and administrators to automate common Kafka administrative operations without interacting directly with Kafka command-line tools.

These APIs can be used to:

- Retrieve cluster information.
- Create and manage Kafka topics.
- Configure topic settings.
- Monitor and manage consumer groups.
- Retrieve topic metadata.
- Manage consumer offsets.
- Perform cluster administration tasks.

The following table provides an overview of the available Kafka Management APIs.

# Kafka Management API Reference

| API | Description |
| --- | --- |
| `/listGroups` | Retrieves all consumer groups available in the Kafka cluster. |
| `/resetOffsets` | Resets the offsets for a consumer group. |
| `/setOffsets` | Updates offsets for one or more topic partitions. |
| `/createTopic` | Creates a new Kafka topic. |
| `/deleteTopic` | Deletes an existing Kafka topic. |
| `/getAllTopicDetails` | Retrieves metadata for all topics in the Kafka cluster. |
| `/getTopicDetails/{topic}` | Retrieves metadata and offset information for a specific topic. |
| `/describeCluster` | Returns broker and cluster metadata. |
| `/describeTopic` | Retrieves configuration details for a specific topic. |
| `/updateTopicConfigs` | Updates the configuration of a Kafka topic. |
| `/getOffsetsForGroup` | Retrieves committed offsets for a consumer group. |
| `/deleteGroup` | Deletes a consumer group from the Kafka cluster. |

## API Contract

### `/listGroups`

**Description**

Returns a list of all consumer groups available in the Kafka cluster.

**Response**

```json
{
  "data": {
    "groups": [
      {
        "groupId": "b8316KqF_M9OwFfFAABXhdfhbu",
        "protocolType": "consumer"
      },
      {
        "groupId": "connect-cluster-sqs",
        "protocolType": "connect"
      }
    ]
  },
  "status": "success",
  "message": "Groups retrieved successfully"
}
```

### `/resetOffsets`

**Request**

```json
{
  "groupId": "my-consumer-group",
  "topic": "postmanValidation"
}
```

**Description**

Resets the offsets for the specified consumer group to the configured target offset. The consumer group must not have any active running instances while performing the reset operation.

**Response**

```json
{
  "status": "success",
  "message": "Offset Reset successfully"
}
```

### `/setOffsets`

**Request**

```json
{
  "topic": "postmanValidation",
  "partitions": [
    {
      "partition": 0,
      "offset": 0
    }
  ]
}
```

**Description**

Updates the offset for the specified topic partition. This operation can also be used to delete records from a topic by specifying the target offset.

**Response**

### `/createTopic`

**Request**

```json
{
  "topicName": "postmanValidation",
  "timeoutTime": 5000,
  "numOfPartitions": 2,
  "configEntries": []
}
```

**Description**

Creates a new Kafka topic. If optional configuration values are not provided, the broker's default configuration is applied.

**Response**

```json
{
  "status": "success",
  "message": "Topic Created Successfully"
}
```

### `/deleteTopic`

**Request**

```json
{
  "topicName": "postmanValidation"
}
```

**Description**

Deletes the specified Kafka topic.

**Response**

```json
{
  "status": "success",
  "message": "Topic deleted successfully"
}
```
