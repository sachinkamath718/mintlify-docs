This FAQ provides answers to common questions about Condense, including its features, deployment options, technical capabilities, industry applications, and support resources.

### 1. What is Condense?

Condense is an all-in-one data streaming platform that simplifies the development, management, and scaling of real-time applications. It integrates fully managed Kafka and deploys via a fully managed BYOC model for seamless operations in cloud environments.

### 2. Who is Condense for?

Condense is designed for enterprises, developers, and data teams looking to streamline real-time data streaming, accelerate application development, and reduce infrastructure management complexity.

### 3. What industries can benefit from Condense?

Condense provides a verticalized ecosystem tailored for industries such as mobility, IoT, travel, hospitality, finance, and more.

### 4. What does fully managed Kafka mean?

Fully managed Kafka automates deployment, scaling, and maintenance of Kafka clusters, ensuring high availability, security, and performance without manual intervention.

### 5. What is BYOC, and how does Condense implement it?

BYOC (Bring Your Own Cloud) allows Condense to operate directly within customer-controlled cloud environments, ensuring complete data sovereignty while automating backend operations.

### 6. How is Condense deployed?

Condense is deployed via cloud marketplaces, including Azure, AWS, and GCP. Users can search for Condense, follow the deployment steps, and customize configurations for their use case.

### 7. What makes the Custom Transformation Framework unique?

The Custom Transformation Framework provides a built-in IDE that allows users to write, test, and deploy complex logic in their preferred programming language, tailored to specific real-time use cases.

### 8. Does Condense support prebuilt transforms and connectors?

Yes, Condense includes a library of industry-specific prebuilt transforms and connectors, enabling faster application development and integration with various data sources.

### 9. What observability features does Condense offer?

Condense includes real-time observability tools to monitor Kafka clusters, track performance metrics, detect anomalies, and optimize workflows seamlessly.

### 10. What are the uptime and reliability guarantees?

Condense offers a 99.95% uptime SLA with multi-cluster support and automated failover mechanisms to ensure uninterrupted streaming.

### 11. Can Condense scale dynamically?

Yes, Condense dynamically scales resources based on workload demands, ensuring consistent performance during peak loads.

### 12. How does Condense handle security?

Condense implements robust security measures, including encryption, role-based access control, and compliance with enterprise and regulatory standards.

### 13. Can Condense help manage cloud costs?

Yes, Condense optimizes resource utilization through dynamic scaling and pay-as-you-go flexibility, reducing Total Cost of Ownership (TCO) for cloud operations.

### 14. How does Condense simplify troubleshooting?

The platform’s built-in Activity Auditor centralizes logs and performance data, helping users quickly identify and resolve issues.

### 15. How does Condense accelerate time-to-market?

Condense provides prebuilt industry-specific tools, a Custom Transformation Framework, and seamless deployment through cloud marketplaces, reducing development cycles and enabling faster go-to-market strategies.

### 16. What business outcomes can Condense deliver?

Condense drives real-time personalization, predictive analytics, operational efficiency, and better customer experiences, helping businesses achieve transformative results.

### 17. Why should I switch to Condense from traditional streaming platforms?

Condense eliminates infrastructure management complexities, provides industry-specific solutions, and accelerates real-time insights, making it more efficient than traditional platforms.

### 18. On which cloud marketplaces is Condense available?

Condense is available on Azure, AWS, and GCP marketplaces, offering seamless deployment options across major cloud providers.

### 19. Is there a free trial available for Condense?

Details about trials and pricing plans can be found on the respective cloud marketplace listing for Condense.

### 20. How do I deploy Condense from a cloud marketplace?

Search for "Condense" in the marketplace, follow the guided deployment steps, and configure the platform according to your needs.

### 21. What kind of support does Condense offer?

Condense provides comprehensive support, including documentation, onboarding assistance, and technical support through its helpdesk and cloud marketplace channels.

### 22. How do I report an issue or request help with Condense?

You can contact the support team directly through the cloud marketplace or Zeliot's support portal.

### 23. Does Condense offer integration guidance?

Yes, detailed integration documentation and tutorials are available to help users connect data sources and build pipelines efficiently.

### 24. What is a workspace in Condense?

A workspace is an isolated environment where you can create, test, and deploy data pipelines.

### 25. Can I create multiple workspaces?

Yes, users can create additional workspaces to organize projects and workflows. Each workspace can have a custom name and description.

### 26. What happens if I delete a workspace?

Deleting a workspace will remove all pipelines, connectors, and transforms associated with it. Ensure you back up any critical data.

### 27. What is the difference between an Application and a Custom Transform?

Applications are development environments where you can build custom logic, while Custom Transforms are finalized, deployable versions of those applications.

### 28. How do I publish an application as a Custom Transform?

Navigate to the application, finalize its logic, and click on "Publish" to convert it into a deployable Custom Transform.

### 29. Can I edit an already published Custom Transform?

Yes, you can create a new version by restoring the application associated with the transform, making changes, and republishing it.

### 30. What types of pre-built connectors are available?

Condense offers Vehicle Telematics, Stream Connectors, and Store Connectors for various real-time data and storage needs.

### 31. How do I configure a pre-built connector?

Navigate to the Connectors section, select a connector, and follow the setup wizard to configure required parameters like topics, credentials, or endpoints.

### 32. Can I create my own custom connector?

Yes, Condense allows users to create and deploy custom connectors tailored to specific data sources or use cases.

### 33. How do I create a data pipeline?

Go to your workspace, select "Add Pipeline," and configure it by adding connectors, transforms, and applications.

### 34. Can I monitor pipeline performance?

Yes, you can use logs and metrics available in the Activity Auditor to monitor pipeline performance and debug issues.

### 35. What happens if a pipeline fails?

You can review the failure logs, identify the issue, and make necessary adjustments to the pipeline or its components.

### 36. What roles are available in Condense?

1. Admin: Full control over the platform.
2. Maintainer: Manage specific workspaces and their resources.
3. Developer: Create and manage applications within assigned workspaces.
4. Viewer: Read-only access to resources.

### 37. Can a user have multiple roles?

No, each user can have only one role within a workspace. However, admins can update roles as needed.

### 38. What happens if an admin deletes their account?

At least one admin must remain in the organization. If the last admin tries to delete their account, the system will prevent it.

### 39. What is the purpose of the Activity Auditor?

The Activity Auditor provides a centralized view of activities and logs across the platform, ensuring transparency and simplifying troubleshooting.

### 40. How far back can I view logs in the Activity Auditor?

Logs are retained for 30 days by default. Contact your administrator for specific retention policies.

### 41. Can I export logs from the Activity Auditor?

Currently, Condense does not support log export functionality.

### 42. How do I connect a code repository?

Go to "Settings \> Code Repository" and follow the prompts to authenticate and link your repository (e.g., GitHub, GitLab, Bitbucket).

### 43. What happens if I disconnect a code repository?

Disconnecting a repository removes its access but does not delete any data. However, linked applications will be removed from the Applications page.

### 44. How do I update my account details?

Go to "Settings \> Personal Details" to update your username, password, or contact information.

### 45. Why can’t I see any logs in the Activity Auditor?

If no logs are displayed, it means no activities have been performed yet. Start using the platform to generate logs.

### 46. What should I do if my development environment expires?

You can restore the expired application or transform it and set a new expiry threshold.

### 47. Why am I unable to deploy a connector or pipeline?

Ensure you have the necessary permissions and verify that all configurations (e.g., credentials, endpoints) are correct.
