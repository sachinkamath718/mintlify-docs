## Architecture Components

Condense employs a three-tier architecture, ensuring clear separation of concerns and promoting efficient communication between components:

![Cover](/images/condense/v2.4.0/getting-started/files%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FrwKRGO3QthZ6EMqqYblg%252Fuploads%252FIbQXqo7bf5fa5cKnTEym%252FCondArchVec2.png)

### Cloud Provider

The cloud marketplace (e.g., Azure, AWS and GCP) serves as the platform for deploying Condense. Users access Condense through the provider's interface.

![Cover](/images/condense/v2.4.0/getting-started/files%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FrwKRGO3QthZ6EMqqYblg%252Fuploads%252FX8wHjqYwVL1I3XroiXJD%252FCondArchVec3.png)

### Zeliot Central Tower

This central hub manages deployments, user authentication, release updates, and monitors the overall health of Condense instances.

![Cover](/images/condense/v2.4.0/getting-started/files%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FrwKRGO3QthZ6EMqqYblg%252Fuploads%252F82dIOJ7tC1Wo34Qs85WI%252FCondArchVec4.png)

### Condense Application

Deployed within the user's cloud environment, this application processes data from mobility devices, performs transformations, and sends it to designated outputs.
